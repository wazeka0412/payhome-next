## 判定方式

全 25 項目を点検し、以下で判定する:

- **S (リリース阻害)**: 1 つでも ❌ があれば NO-GO (リリース延期)
- **A (必達)**: 2 つ以上 ❌ があれば NO-GO
- **B (望ましい)**: 情報として記録、リリース自体は阻害しない

## S 項目 (リリース阻害・全て ✅ 必須)

### S1. 本番環境

- [ ] Vercel 本番プロジェクトが payhome.jp ドメインで疎通
- [ ] SSL 証明書が有効 (Vercel 自動)
- [ ] 環境変数 14 個が全て Production に設定済み
- [ ] `NEXTAUTH_SECRET` が本番用に新規生成されている (dev と別)
- [ ] Supabase 本番プロジェクトに接続済み
- [ ] マイグレーション `001`〜`004` が適用済み
- [ ] 工務店 10 社の初期データが `builders` テーブルに投入済み
- [ ] 見学会イベントが 5 件以上 `events` テーブルに投入済み (5月以降)

### S2. 主要機能

- [ ] 17 公開ページ + 5 補助ページが全て 200 で表示される
- [ ] 13 以上の非公開パスが全て 404 を返す
- [ ] `/signup` で新規会員登録が成功する
- [ ] 登録後 `/welcome` に 200 でリダイレクトされる (← 重要バグ履歴あり)
- [ ] 匿名 ID (`anonymous_id`) が登録時に `user_id` に紐付く
- [ ] AI 診断 10 問が完走し、結果が `diagnosis_sessions` に保存される
- [ ] 見学会予約フォームが送信でき、`leads` に保存される
- [ ] 見学会予約通知が工務店に届く (メール + ダッシュボード)

### S3. セキュリティ

- [ ] Cloudflare Turnstile が signup/contact/event booking で有効
- [ ] 管理画面 `/admin/*` が admin ロール以外で 404 または 403
- [ ] 工務店ダッシュボード `/dashboard/builder/*` が builder ロール以外で 404 または 403
- [ ] PII (メール・電話) が HTTPS 以外で送信されない

### S4. 監視

- [ ] Sentry がエラーを補足する (Test エラーで確認済み)
- [ ] Vercel Analytics が有効
- [ ] GA4 が `diagnosis_start` / `signup_complete` / `event_booking` イベントを記録
- [ ] 代表・PM・エンジニアリーダーの緊急連絡先が共有済み

## NO-GO 時の対応

1. 延期日を決定 (通常 1 週間以内)
2. ❌ 項目の原因分析と修正計画を作成
3. 再判定日を設定
4. 10 社・社内・外部関係者に延期を連絡
5. リリース前チェックリストを再実施
