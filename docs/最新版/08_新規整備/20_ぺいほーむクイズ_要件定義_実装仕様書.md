# ぺいほーむクイズ 要件定義 + 実装仕様書

最終更新: 2026-04-12
対象: PM / 開発 / デザイン / 営業
ステータス: **MVP v1 実装済み (Phase 1)**

---

## 1. 概要と目的

ぺいほーむクイズは **「家づくり理解度を可視化して、次の最適行動へ導く教育導線」** である。

### AI家づくり診断との違い

| | ぺいほーむクイズ | AI家づくり診断 |
| --- | --- | --- |
| 目的 | 家づくりの **理解度** を可視化 | 自分に **合う家・工務店** を整理 |
| 結果 | レベル判定 + 苦手カテゴリ | ユーザータイプ + 推薦工務店 3 社 |
| 位置づけ | 教育 → 核ループへの合流 | マッチング → 見学会予約 / 無料相談 |
| 核ループとの関係 | 補助導線 (核ループの手前) | 核ループの中核入口 |

クイズは核ループ (動画 → AI 診断/無料相談 → 工務店 → 見学会予約) を壊さず、核ループへ滑らかに接続する。

---

## 2. 想定ユーザー

- **家づくり初心者**: 「何から始めればいいかわからない」人。クイズで理解度を可視化し、AI 診断や動画に誘導
- **比較検討中のユーザー**: 基礎知識は持っているが一部不足がある人。苦手カテゴリを補強して次に進む
- **家づくり経験者**: 知識は十分な人。見学会予約や無料相談で具体的に前に進める

---

## 3. ユーザーストーリー

- 初心者として: 「10問のクイズで自分の理解度を知り、何が足りないかわかった。次にAI診断を受けて方向性を整理しよう」
- 中級者として: 「住宅ローンの知識が弱いことがわかった。動画や事例で勉強してから工務店を比較しよう」
- 上級者として: 「知識は十分だと確認できた。見学会を予約して実物を見に行こう」

---

## 4. MVP スコープ

### 含むもの (Phase 1)

- `/quiz` 単一ページ (イントロ → 10 問回答 → 結果画面)
- 4 カテゴリ × 10 問 (建売と注文住宅の違い / 住宅ローン / 家づくりの流れ / 平屋の特徴)
- 3 段階レベル判定 (初心者 / 中級者 / 上級者)
- カテゴリ別正答率 + 苦手カテゴリ表示
- レベル別 CTA 出し分け (初心者→AI診断・無料相談、中級者→事例・工務店、上級者→見学会・無料相談)
- 非会員でも利用可
- 会員は結果を localStorage + API 保存
- 正答ごとに解説表示
- Header「サービス」ドロップダウンに追加
- `/welcome` の会員機能カードに追加

### 含まないもの (Phase 2 以降)

- カテゴリ別の個別クイズ (建売クイズ / ローンクイズ 等)
- 難易度別クイズ (初級 / 中級 / 上級)
- ランキング / ポイント
- クイズ結果を AI 診断に連携 (将来統合)
- 工務店向けのリードスコアリングへの統合
- マイページでのクイズ履歴一覧
- 管理画面でのクイズ問題 CRUD
- Supabase quiz_sessions テーブル (MVP は localStorage のみ)

## 5. ページ構成への位置づけ

### グローバルナビ

「サービス」ドロップダウン:
1. AI家づくり診断 (10問で自分に合う家を提案)
2. **ぺいほーむクイズ (家づくり理解度をチェック)** ← 新規追加
3. 無料住宅相談 (専門スタッフに相談)

### TOP ページ

TOP のメインセクションには入れない (核ループ集中)。フッター「サービス」列に追加。

### /welcome

会員登録直後の会員機能カード (4 枚グリッド) に追加:
- 動画で比較
- 事例を全件閲覧
- **家づくり理解度をチェック** ← 新規追加
- 連絡条件を設定

### sitemap.xml / robots.txt

`/quiz` を公開パスとして追加。middleware には含めない (公開)。

## 6. 技術実装

### ファイル構成

```
src/
├── lib/quiz-data.ts                    — 問題定義 + レベル判定ロジック
└── app/(user)/quiz/page.tsx            — クイズ UI (イントロ/回答/結果)
```

### 問題定義 (`src/lib/quiz-data.ts`)

- `QuizQuestion` 型: id, category, question, options, correctIndex, explanation, difficulty
- `QuizResult` 型: score, total, level, categoryScores, weakCategories
- `QUIZ_QUESTIONS`: 10 問の配列
- `calculateQuizResult(answers)`: 全体スコア + レベル + カテゴリ別正答率 + 苦手カテゴリを算出
- `CATEGORY_LABELS`: 4 カテゴリの日本語ラベル
- `LEVEL_LABELS`: 3 レベルのラベル + 説明文 + カラー

### クイズ UI (`src/app/(user)/quiz/page.tsx`)

3 フェーズの `useState`:
1. **intro**: 開始画面 (カテゴリ説明 + 所要時間 + AI診断との違い + 開始ボタン)
2. **quiz**: 10 問を順次表示 (プログレスバー + 選択肢ボタン + 正誤表示 + 解説 + 次へ)
3. **result**: 結果画面 (レベルカード + カテゴリ別バー + レベル別CTA + 会員登録訴求)

### データ永続化

#### MVP (Phase 1)

- `localStorage.setItem('payhome_quiz_result', JSON.stringify(data))` でクライアント側に保存
- 会員の場合は `POST /api/quiz/result` にも送信 (Phase 1 では API は存在しないが、エラーをキャッチして無視する設計)

#### Phase 2 以降

```sql
-- migration: 006_add_quiz_sessions.sql
CREATE TABLE quiz_sessions (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id uuid REFERENCES users(id),
  anonymous_id text,
  answers jsonb NOT NULL,  -- [{questionId, selectedIndex}]
  result jsonb NOT NULL,   -- {score, total, level, categoryScores, weakCategories}
  completed_at timestamptz DEFAULT now(),
  created_at timestamptz DEFAULT now()
);

CREATE INDEX idx_quiz_sessions_user ON quiz_sessions(user_id);
CREATE INDEX idx_quiz_sessions_anon ON quiz_sessions(anonymous_id);
```

## 7. 会員登録との関係

- クイズ自体は **非会員でも利用可能** (ログイン不要)
- 会員登録すると:
  - クイズ結果が API 経由で永続保存される (Phase 2)
  - マイページでクイズ履歴を閲覧可能 (Phase 2)
  - AI 診断結果 + クイズ結果を組み合わせた推薦精度向上 (Phase 3+)
- 結果画面の会員訴求文言:
  > 「会員登録で、クイズ結果を保存できます。理解度の記録・AI診断結果の保存・お気に入り整理・事例全件閲覧まで、家づくりが一気に進む会員機能が使えるようになります。」

## 9. KPI

### Phase 1

| 指標 | 目標 | 備考 |
| --- | --- | --- |
| クイズ開始数 | 月 100 回 | (TOP / Header / welcome からの流入) |
| クイズ完走率 | 70% 以上 | (10 問全回答) |
| 結果画面 CTA クリック率 | 30% 以上 | (レベル別 CTA のいずれか) |
| 結果画面 → AI 診断遷移率 | 15% 以上 | (初心者・中級者) |
| 結果画面 → 見学会予約遷移率 | 10% 以上 | (上級者) |
| 結果画面 → 会員登録率 | 5% 以上 | (非会員のみ) |

### 計測方法

GA4 イベント:
- `quiz_start`: クイズ開始ボタンクリック
- `quiz_answer`: 各問題の回答 (question_id, is_correct, category)
- `quiz_complete`: 全問回答完了 (score, level)
- `quiz_cta_click`: 結果画面 CTA クリック (cta_id, level)

## 11. 事業・営業面への効果

### 教育導線の強化

ぺいほーむは「単なる送客業者」ではなく、ユーザーを教育し理解度を高めた状態で送客する。クイズはその教育プロセスの可視化ツール。

### 会員価値の強化

クイズ結果の保存・継続学習・理解度の蓄積が会員機能の一部になることで、「家づくりが進む」会員訴求に厚みが出る。

### リード精度向上 (Phase 3+)

クイズの回答データ (理解度 + 苦手カテゴリ) を AI 診断や無料相談の前提情報として渡すことで、工務店への送客精度が上がる。

### 成約報酬 50万円の付加価値補強

「教育 → 理解度可視化 → 最適導線」というプロセスが、成約報酬 50万円の 6 つの付加価値のうち「事前教育 (Pre-Education)」を直接的に強化する。
