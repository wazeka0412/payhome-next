# 成約報告・証憑提出・請求フロー説明書

最終更新: 2026-04-12
対象: 工務店様・ぺいほーむ経理・ぺいほーむ PM
適用開始: 2026-09-01 (Phase 3 より本格運用)

---

## 1. 全体フロー図

```
成約 (建築請負契約書署名)
   ↓ (7営業日以内)
工務店: 成約報告フォーム送信 + 証憑提出
   ↓
ぺいほーむ: lead_source 自動判定 + 対象リード確認
   ↓
ぺいほーむ: 月末締めで集計
   ↓ (翌月初3営業日)
ぺいほーむ: 請求書発行 (PDF)
   ↓
ぺいほーむ: メール送信 + 工務店ダッシュボードに通知
   ↓ (30日以内)
工務店: 振込支払
   ↓
ぺいほーむ: 入金確認 → leads.status = 'paid'
```

---

## 2. 工務店側の手順

### 2.1 成約発生時 (建築請負契約書の署名日)

1. 建築請負契約書に双方が署名
2. 契約書の写しを PDF 化

### 2.2 報告 (署名日から 7 営業日以内)

1. 工務店ダッシュボードにログイン (`/dashboard/builder/leads`)
2. 該当ユーザーの行を開き「成約報告」をクリック
3. 以下を入力:
   - 成約日 (建築請負契約書の署名日)
   - 契約額 (税抜・本体価格、追加工事・オプション除外)
   - 備考 (キャンセル可能性等)
4. 証憑を PDF でアップロード (建築請負契約書または見積書のいずれか)
5. 「報告する」をクリック

### 2.3 メール経由の場合 (ダッシュボード操作が困難な場合)

`support@payhome.jp` に以下を送信:

```
件名: 成約報告 [工務店名] - [ユーザー氏名]

- 工務店名:
- 工務店担当者:
- 成約ユーザー氏名:
- ユーザーID または lead_id:
- 成約日 (YYYY-MM-DD):
- 契約額 (税抜・本体価格):
- 備考:
- 証憑添付: 建築請負契約書.pdf
```

## 3. ぺいほーむ側の処理

### 3.1 報告受領 (当日)

1. 経理 + PM が Slack `#payhome-leads-closed` で通知を受ける
2. PM が 48 時間以内に `lead_source` を確認

### 3.2 対象判定 (報告受領から 48 時間以内)

ぺいほーむ側で以下を確認:

- `leads.lead_source` が `consultation` / `diagnosis` / `member` のいずれか → **対象**
- `leads.lead_source` が `direct` → **対象外** (成果報酬発生せず、工務店に通知)
- リード発生日から成約日までが 6 ヶ月以内か確認
- 重複成約がないか確認 (過去の成約報告との突合)

### 3.3 対象外の場合

以下テンプレートで工務店に通知:

```
件名: 成約報告の対象判定結果 [ユーザー名]

お世話になっております。

先日ご報告いただいた成約につきまして、確認したところ、以下の理由により成果報酬の対象外となります。

対象外理由:
- [ ] リードの発生経路 (lead_source) が対象外 (direct 扱い)
- [ ] リード発生から成約まで 6 ヶ月を超過
- [ ] その他: ___

成果報酬の請求は行いませんが、本成約を踏まえて、Phase 2 以降の撮影プランや月額掲載プランのご相談も承ります。

ご不明点は support@payhome.jp までご連絡ください。

株式会社wazeka ぺいほーむ事業部
```

### 3.4 対象の場合

- `leads.status = 'closed_won'` に更新
- `leads.contract_amount` に契約額を記録
- `leads.contract_signed_at` に成約日を記録
- `leads.success_fee_amount` に契約額 × 3% を記録
- 証憑を `docs/経理/証憑/[年月]/[工務店名]/[ユーザー名].pdf` に保管

### 3.5 月末集計 (毎月月末)

1. 当月発生した対象成約を全件リスト化
2. 工務店ごとに合計成果報酬額を算出
3. Excel シート `請求・P&L管理.xlsx` の `① 月次請求台帳` に記録

### 3.6 請求書発行 (翌月初 3 営業日)

1. freee の請求機能で請求書を作成
2. PDF で出力
3. 工務店にメール送信 + 工務店ダッシュボードに通知

### 請求書の記載項目

- 発行者 (甲): 株式会社 wazeka
- 宛先 (乙): 工務店名・担当者
- 請求日
- 支払期限 (発行日から 30 日後)
- 明細:
  - 成果報酬 (2026/09 分) ○○様 - ユーザー A 成約: ¥900,000
  - 成果報酬 (2026/09 分) ○○様 - ユーザー B 成約: ¥1,500,000
  - 小計: ¥2,400,000
  - 消費税: ¥240,000
  - 合計: ¥2,640,000
- 振込先: 株式会社 wazeka の指定口座

### 3.7 入金確認 (翌々月月末まで)

- 経理がネットバンキングで入金確認
- `請求・P&L管理.xlsx > ② 入金管理` に入金日・入金額を記録
- `leads.status = 'paid'` に更新
- 支払遅延があれば PM にエスカレーション

## 6. 重要な注意事項

### 6.1 秘密保持

- 契約書・証憑に含まれる個人情報は厳重管理
- 第三者に開示しない (法的要請を除く)
- 経理以外のスタッフは証憑にアクセスできない

### 6.2 報告漏れ・虚偽報告の対応

詳細は `10_成果報酬3%契約別紙.md > 第 8 条` 参照。

- 報告漏れ発見時: 乙は速やかに追加支払
- 虚偽報告発見時: 乙は正しい契約額の成果報酬 + **過少申告額の 2 倍のペナルティ**
- 反復的な不正: 契約即時解除 + 損害賠償請求

### 6.3 契約解除後の扱い

契約解除後も、解除日前に発生した対象リードからの成約は、解除日から 6 ヶ月以内に限り、成果報酬の対象となる。

---

## 7. エスカレーション基準

- 入金遅延 30 日以上 → PM → 代表
- 成約報告の内容で揉めた → PM → 代表
- 契約額の過少申告疑惑 → 代表 + 外部弁護士
- 重大な不正発見 → 代表 + 外部弁護士 + 監査役
