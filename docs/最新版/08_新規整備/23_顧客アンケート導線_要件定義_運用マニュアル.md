# 顧客アンケート導線 要件定義・運用マニュアル

最終更新: 2026-04-12
対象: PM / CS / 営業 / エンジニア / 経理

---

## 1. 目的

ぺいほーむは単なる送客元ではなく、家づくりを伴走支援する第三者。
会員登録後〜成約まで継続接点を持ち、以下を実現する。

- 温度感・進捗の把握（工務店申告に依存しない）
- 商談化兆候の早期検知
- 成果報酬50万円の対象判定の補強
- 顧客満足度の向上

---

## 2. 全体フロー

```
会員登録 → /welcome（伴走メッセージ）
    ↓
見学会予約 → 予約完了画面（アンケート導線）
    ↓
予約後アンケート回答 → 特典送付
    ↓
見学会参加
    ↓
見学後アンケート（2-5日後送信）→ 特典送付
    ↓
商談・検討期間
    ↓
成約報告（顧客 + 工務店双方から）→ 証憑確認 → ギフト
```

---

## 3. フェーズ別設計

### A. 会員登録後（/welcome）

| 項目 | 内容 |
|------|------|
| 画面 | /welcome |
| 変更内容 | 伴走メッセージ追加「あなたの家づくりの味方です」 |
| 目的 | 送客業者ではなく、第三者の伴走役であることを印象付け |
| 次のアクション誘導 | AI診断 / クイズ / 事例保存 / 無料相談 |

### B. 見学会予約後アンケート

| 項目 | 内容 |
|------|------|
| 画面 | /survey/post-booking |
| トリガー | 見学会予約完了画面のCTA |
| 質問数 | 5問 |
| 所要時間 | 約1分 |
| 特典 | 見学会チェックリスト / 家づくり失敗回避ガイド / 平屋比較シート |

**質問内容:**

| # | 質問 | 形式 | 選択肢 |
|---|------|------|--------|
| Q1 | 現在の検討度合い | 単一選択 | 情報収集段階 / 比較検討中 / かなり具体的 |
| Q2 | 現在何社くらい比較 | 単一選択 | まだ0社 / 1〜2社 / 3社以上 |
| Q3 | 今いちばん不安なこと | 複数選択 | 予算・ローン / 間取り / 土地 / 工務店選び / スケジュール / その他 |
| Q4 | 見学会で知りたいこと | 複数選択 | 施工品質 / 間取り・広さ / 価格感 / スタッフ対応 / 設備・性能 |
| Q5 | 家づくり情報を受け取るか | 二択 | はい / 今はいいです |

### C. 見学会後アンケート

| 項目 | 内容 |
|------|------|
| 画面 | /survey/post-visit |
| トリガー | 見学日から2〜5日後にメール/LINEで送信 |
| 質問数 | 5問 + 相談希望チェック |
| 特典 | 工務店比較テンプレ / 間取り質問リスト / 比較表 |

**質問内容:**

| # | 質問 | 形式 |
|---|------|------|
| Q1 | 見学会の満足度 | 5段階 |
| Q2 | 工務店の印象 | 4段階 |
| Q3 | 次回面談予定 | 単一選択 |
| Q4 | 他社と比較中か | 単一選択 |
| Q5 | 今いちばんの悩み | 自由記述 |
| + | 無料相談希望 | チェック |

### D. 成約報告

| 項目 | 内容 |
|------|------|
| 画面 | /mypage/conversion-report |
| トリガー | 見学後1-6ヶ月でフォローメール / マイページ常設リンク |
| 内容 | 成約有無 / 成約日 / 成約先 / ギフト希望 / メッセージ |
| ギフト | 証憑確認後に送付（新生活ギフト等） |

## 4. 成果報酬50万円判定への反映

| アンケート | 成果報酬判定での扱い |
|-----------|---------------------|
| 予約後アンケート回答 | 教育導線通過・継続接点の証拠 |
| 見学後アンケート回答 | 商談化兆候把握の証拠 |
| 顧客成約報告 | 成約把握の補助（工務店報告との二重化） |

**不一致時の運用:**
- 顧客が成約報告 → 工務店未報告: PMが工務店に確認連絡
- 工務店が成約報告 → 顧客未報告: 証憑で確認、顧客への確認は任意
- 双方報告あり: 最も信頼性の高い証拠として扱う

## 5. 運用マニュアル

### 日次

| 担当 | アクション |
|------|-----------|
| CS | 新規アンケート回答を確認（/admin または Supabase） |
| CS | 相談希望チェック者に24h以内に連絡 |

### 週次

| 担当 | アクション |
|------|-----------|
| PM | 見学後2-5日経過者リストを抽出 → 見学後アンケート送信 |
| PM | アンケート回答率を集計 |
| 営業 | 商談化兆候ありの案件を工務店に共有 |

### 月次

| 担当 | アクション |
|------|-----------|
| PM | アンケート結果サマリーを月次レポートに反映 |
| PM | 成約報告の顧客/工務店突合を実施 |
| 経理 | 成約報告案件の証憑確認 → 請求処理 |

## 6. プレゼント運用ルール

| フェーズ | 種別 | 付与条件 | 付与方法 |
|---------|------|---------|---------|
| 予約後 | デジタル | アンケート完了 | 自動メール送信 |
| 見学後 | デジタル | アンケート完了 | 自動メール送信 |
| 成約時 | 物理ギフト | 証憑確認完了 | PMが手配（1万円以下） |

## 7. DB設計

テーブル: `surveys`（005_surveys.sql）

| カラム | 型 | 用途 |
|--------|-----|------|
| id | UUID | PK |
| user_id | UUID | 会員ID |
| survey_type | TEXT | post_booking / post_visit / conversion_report |
| event_id | TEXT | 見学会ID |
| builder_name | TEXT | 工務店名 |
| responses | JSONB | 回答データ |
| conversion_date | DATE | 成約日 |
| conversion_company | TEXT | 成約先 |
| evidence_submitted | BOOLEAN | 証憑提出済み |
| gift_requested | BOOLEAN | ギフト希望 |
| gift_sent | BOOLEAN | ギフト送付済み |
