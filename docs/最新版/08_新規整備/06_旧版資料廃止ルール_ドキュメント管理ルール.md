# 旧版資料廃止ルール & ドキュメント管理ルール

最終更新: 2026-04-12
対象: 全スタッフ
適用開始: 2026-04-12

---

## 1. 目的

ぺいほーむ事業で使用するドキュメントは料金モデル・MVP 方針・運用フローの更新が頻繁であり、**旧版が誤って参照されると事故につながる**。本書はこれを防止するための最小限のルールを定義する。

---

## 2. 最新版の原則

**最新版の保管場所は `docs/最新版/` 配下のみ** とする。

```
docs/最新版/
├── README.docx                          ← 全体インデックス
├── 01_エンジニア共有用/                   ← 技術仕様
├── 02_管理シート/                         ← 管理表
├── 03_営業関係/                           ← 営業資料
├── 04_経理関係/                           ← 請求・P&L
├── 05_契約書/                             ← 契約テンプレート
├── 06_事業計画・戦略/                     ← 経営層向け
├── 07_業務運用ルール/                     ← オペレーション
└── 08_新規整備/                           ← 2026-04-12 以降の新規追加資料
```

**`docs/最新版/` 以外のファイル (`docs/01_〜docs/99_` の直接配下) は過去履歴とし、アクティブ参照には使わない。**

---

## 3. 旧版マーキングルール

旧版ファイルを発見したら、以下のいずれかを行う:

### A. 削除できるもの (最新版で完全置換されている)

即座に削除する。Git 履歴に残る。

### B. 履歴として残すべきもの

ファイル名に以下のサフィックスを追加する:

- `_pre-MVP版.xxx` — MVP 確定前の資料
- `_v1_archive.xxx` — v1 料金モデル時代の資料
- `_旧版.xxx` — 廃止されたが参照履歴がある資料

### C. 複雑で置換が難しい docx/pptx (旧版ドキュメント)

文書の冒頭に以下の警告バナーを追記する:

```
⚠️ 【警告: 旧版・参照禁止】
この資料は旧方針 (v1) で作成されています。
現行の v2 方針は docs/最新版/08_新規整備/00_現行方針1枚まとめ.md を参照してください。
最新の営業説明資料は docs/最新版/03_営業関係/ を参照してください。

作成日: YYYY-MM-DD
廃止日: 2026-04-12
代替資料: (新しいファイルへのリンク)
```

## 5. ドキュメント命名規則

### 新規作成時

- 日本語の意味が明確になる名前を使う
- バージョン番号は原則付けない (Git 履歴で管理)
- 例外的にリリース版を示すときは `_MVP版.xxx` または `_v2版.xxx`

### サフィックス

| サフィックス | 用途 |
| --- | --- |
| `_MVP版` | 現行 MVP (2026-05-01) 対応 |
| `_v2版` | 料金モデル v2 対応 |
| `_pre-MVP版` | MVP 確定前の旧版 |
| `_旧版` | 廃止された旧版 |
| `_v1_archive` | v1 時代のアーカイブ |
| `_draft` | ドラフト・未確定 |

## 7. 編集ルール

### md ファイル (技術仕様・新規整備等)

- 直接編集 OK
- コミットメッセージに変更理由を記載
- 大きな変更はレビューを通す

### xlsx ファイル

- 直接編集 OK
- 構造変更 (シート追加・列追加) は `scripts/create-latest-*.py` を更新して再生成
- 実績値の入力は直接編集

### docx ファイル

- 直接編集は避ける
- 更新時は source md (`/tmp/payhome-vXX/` 相当) を作成 → `scripts/convert-latest-md-to-docs.py` で docx 化
- この手順を取らない場合はコミットせず、PM に相談

### pptx ファイル

- 直接編集は避ける
- 大きな変更は source md から再生成
- 微修正 (文言のみ) は直接編集可

## 8. 監査ルール

月次で以下を監査:

- `docs/最新版/` に料金 v1 の記述 (1.5%, 永続無料, フリープラン 等) が残っていないか → `scripts/audit-latest-v2.py` 実行
- `docs/最新版/` に `資料請求` が主要導線として残っていないか → grep `資料請求.*主`
- 旧版ファイルに警告バナーが付いているか → 目視確認
- 最新版 README と各 00_README が実ファイル一覧と一致しているか

---

## 9. 誤利用時の対応

- スタッフが誤って旧版資料を工務店・顧客に送付した場合 → 速やかに訂正連絡
- 再発防止策を会議で共有
- 発見されなかった期間が長い場合は影響範囲を調査

---

## 10. 最新版 README の役割

`docs/最新版/README.docx` は全カテゴリへの入口。新規参加者はここから読み始める。

- 収益モデル v2 のサマリー
