# UAT 受け入れテスト仕様書

最終更新: 2026-04-12
対象リリース: 2026-05-01 MVP v2
実施者: PM + 代表 + エンジニア + 営業リーダー (4 名で分担)

---

## 1. 目的

本番リリース前に、MVP v2 の主要フロー・主要画面・主要操作が **ユーザー目線で破綻なく動作する** ことを確認する。

---

## 2. テスト範囲

### A. ユーザー向け主要フロー (核ループ)

1. TOP → 動画詳細 → 工務店詳細 → 見学会予約 → 予約完了 → (会員登録)
2. TOP → AI家づくり診断 (10問完走) → 結果画面 → 推薦工務店 → (会員登録) → `/welcome`
3. TOP → 無料相談フォーム → 送信完了
4. TOP → 事例ライブラリ → 事例詳細 (5 件目まで) → 6 件目で会員ゲート
5. TOP → AIチャット → 質問 → 応答 → 見学会予約誘導

### B. 会員機能

1. 会員登録 (メール・Google・Apple 各 OAuth)
2. `/welcome` ページの診断済み/未実施の出し分け
3. マイページでお気に入り追加・削除
4. `/mypage/contact-preferences` で連絡希望条件を保存
5. `/mypage/catalog` でデジタルカタログ閲覧 (診断完了必須)
6. ログアウト → 再ログイン → 状態復旧

### C. 工務店ダッシュボード

1. `/dashboard/builder` でリード一覧表示
2. `/dashboard/builder/events` で見学会イベント CRUD
3. `/dashboard/builder/leads` で個別リード詳細表示
4. `/dashboard/builder/profile` で掲載情報編集・保存

### D. 管理画面

1. `/admin/dashboard` で集計表示
2. `/admin/leads` でリード一覧・絞り込み
3. `/admin/builders` で工務店一覧・CRUD
4. `/admin/events` でイベント一覧・承認

### E. エッジケース

1. Supabase 接続失敗時のローカルフォールバック動作
2. 非会員で `/mypage` アクセス → ログインガード
3. 非会員で `/mypage/catalog` → 会員登録ガード
4. 非会員で 6 件目の事例詳細 → 会員ゲート
5. 40+ 非公開パス全てが 404 を返す
6. `/welcome` が登録完了後のリダイレクト先として 200 を返す

## 3. テストケース (カテゴリ別)

### TC-001: TOP → AI診断 → 会員登録 → `/welcome` フロー

**前提条件**: 非会員 (シークレットウィンドウ)、Cookie クリア

**手順**:
1. `https://payhome.jp/` にアクセス
2. Hero の「AI診断をはじめる」をクリック
3. 10問の質問に全て回答 (1 問ずつ)
4. 結果画面で「会員登録で結果を保存」をクリック
5. `/signup` で登録情報入力 (テストメール) → 送信
6. `/welcome` にリダイレクトされる

**期待結果**:
- 各画面が 1 秒以内に表示される (体感)
- 診断結果が `/welcome` で推薦 3 社として表示される
- 各推薦カードに「見学会予約」(主) と「工務店を見る」(副) の 2 ボタンがある
- 「まだ決められない時は」無料相談副導線が表示される
- カタログ折りたたみ (BONUS) が表示される

**判定**: [ ] 合格 / [ ] 不合格 / [ ] NA

**判定**: [ ] 合格 / [ ] 不合格

## 5. 不合格時のフロー

1. 不合格を検出した時点で、PM に即時エスカレーション
2. PM が重大度を判定 (P0: リリース阻害 / P1: 要修正・リリース可 / P2: 次週改善)
3. P0 はリリース延期も検討
4. P1 は開発チームが 48 時間以内に修正 → 再テスト
5. P2 はチケット化して次週以降

---

## 6. UAT 完了の定義

- 10 ケース全て合格 (または P2 以下の不合格のみ)
- 本書が全担当者の署名付きで完了報告される
- PM がリリース Go 判定を出す (`リリースGo-NoGo判定シート.md` 参照)
