# 2026年5月リリース前 専門家3名 徹底レビュー

**レビュー日**: 2026/04/11
**対象**: ぺいほーむポータルサイト（2026/05/01 正式リリース予定）
**レビュアー**:
- **専門家A**：住宅ポータル UX 設計者（SUUMO・LIFULL HOME'S 出身の想定）
- **専門家B**：スタートアップ CTO / プロダクトリライアビリティ（B2B SaaS 10年）
- **専門家C**：地域工務店 経営コンサルタント（船井総研 住宅事業部 想定）

---

## 📋 レビュー方法

3名の専門家が、実際の `http://localhost:3000` にアクセスし、以下を検証:
1. ユーザー側サイト（トップ〜診断〜予約フロー）
2. 工務店ダッシュボード（/dashboard/builder）
3. 管理者画面（/admin）
4. 技術基盤（API・データ永続化・セキュリティ）
5. コンテンツ品質（動画・記事・施工事例）
6. 料金設計と運用フロー

各カテゴリで **「リリース可否」「必須改善」「推奨改善」「将来課題」** の4段階で評価。

## 👤 専門家A：住宅ポータル UX 設計者のレビュー

> 「ポータルとしての一貫性・CVR設計・ユーザー心理を厳しく評価します」

### 🟢 高く評価できる点

#### 1. **Smart Match / 連絡希望プロファイルは業界の一歩先を行っている**
10年前からSUUMOでやりたかったけれど、実現できなかったのがこれです。業界の構造的課題である「しつこい営業問題」を**技術的に解決**しようとしている点は、SUUMOやLIFULL HOME'Sがいまだに実装できていない強力な差別化要因です。

#### 2. **AI診断から見学会予約までの導線が自然**
10問2分のAI診断は、ユーザー心理的に「面倒」と感じさせない絶妙な質問数です。診断後に相性の良い工務店3社を提示する画面は、選択肢過多による意思決定疲労を避けられています。

#### 3. **動画という強力な集客資産**
SUUMOは広告費で集客していますが、ぺいほーむはYouTubeのオーガニック流入で集客できている。これは構造的に持続可能で、CAC（顧客獲得コスト）を極小化できる唯一の方法です。

### 🔴 リリース前に必須改善 (3項目)

#### 必須改善 1: **ファーストビューの訴求が弱い**
**問題**: 現状のトップページは「ぺいほーむ住宅ポータル」という抽象的なブランド訴求のみ。ユーザーが「何をしてくれるサイトか」を3秒で理解できません。

**対策**: ヒーローエリアに以下を明示してください：
- 「AIが2分で、あなたに合う工務店を3社提案」
- 「登録・相談・資料請求 全て無料」
- 「九州の平屋なら、ぺいほーむ」

ボタンは「無料でAI診断を始める」を最大サイズで1つだけ配置。「動画を見る」などの二次ボタンは小さく。

#### 必須改善 2: **スマートフォンでの表示最適化の検証不足**
**問題**: 住宅系ユーザーの70%以上がスマホから来訪しますが、現状のレイアウト検証がPC中心と推察されます。特に見学会予約フォームの入力欄がスマホで窮屈に見える可能性があります。

**対策**: iPhone SE (375px幅) と iPhone 14 Pro Max (430px幅) の2サイズで全画面を検証。入力欄は`h-12`以上、ボタンは`h-14`以上で再調整。

#### 必須改善 3: **個人情報の扱いに関する明示不足**
**問題**: 見学会予約フォームで入力する個人情報が「どの工務店に渡されるのか」「何に使われるのか」が分かりにくい。特にスマートマッチで複数社にデータが渡る場合、ユーザーの不安が残ります。

**対策**: 予約フォーム直前に、以下を明示するチェックボックスを追加:
- [ ] 〇〇工務店様にお名前・連絡先・希望条件をお伝えすることに同意します
- [ ] 第三者への提供はされないことを了承しました

### 🟡 推奨改善 (リリース後3ヶ月以内)

1. **施工事例ページの「比較カート機能」** — 気になった事例を複数ブックマークして比較できる機能
2. **お客様の声の動画化** — 文字の体験談より動画の体験談の方が信頼度が圧倒的に高い
3. **AI診断結果の「その後」への動線** — 診断結果を見ただけで離脱するユーザーが多いので、「この結果をPDFで保存」「3社に一括資料請求」などの次アクションを用意
4. **リードタイム表示** — 見学会予約から実際の見学までの日数目安を表示（例：「最短3日後から見学可能」）

### 🔵 将来課題 (1年以上先)

- マイページ内の「家づくりジャーニー管理」（診断→事例閲覧→見学会→契約 の進捗を時系列で管理）
- チャットボット（AI）によるFAQ 24時間対応
- 住宅ローン事前審査との連携（LIFULL HOME'S が最近導入）

### 💬 総評
> 「リリース可否：**条件付き可**。上記『必須改善3項目』を完了すれば、5月リリースは問題ありません。
> ポテンシャルは極めて高く、私が SUUMO 時代に『やりたかった』ことを実現しています。特に Smart Match と AI診断は、業界平均のCVR (1-2%) を **3-5%まで引き上げる可能性** があります。」

## 🔧 専門家B：スタートアップ CTO のレビュー

> 「技術的な安定性・セキュリティ・運用可能性を厳しく評価します」

### 🟢 高く評価できる点

#### 1. **API とコンテンツのフォールバック設計が堅牢**
Supabaseが接続できない場合に `.local-data` のJSONフォールバックで動作する設計は、スタートアップとして非常に現実的です。リリース初期に Supabase の設定ミスで全停止するリスクを回避できています。

#### 2. **管理画面（/admin）の CMS 機能が充実**
15種類のコンテンツタイプが管理画面から CRUD できる状態になっているのは驚きです。これは通常、リリース後3-6ヶ月かけて実装する範囲です。

#### 3. **Next.js 16 の Server Components を適切に活用**
モダンなフロントエンド基盤で、パフォーマンス・SEO の両面で有利。中長期的なメンテナンス性も高い。

### 🔴 リリース前に必須改善 (5項目)

#### 必須改善 1: **エラーハンドリングの網羅性不足**
**問題**: `/api/leads` のPOSTは try/catch で守られていますが、AI診断の結果保存、成約トラッキング、支払い記録などの「失敗したら売上を逸失する」重要処理のエラーハンドリングが不十分に見えます。

**対策**:
- 全APIで Sentry（またはLogtail）などのエラー追跡ツールを導入
- 重要な処理（予約・成約・支払い）は失敗時にSlackやメールで即時通知
- DB書き込み失敗時は自動リトライ（最大3回）

#### 必須改善 2: **個人情報の暗号化・バックアップ体制**
**問題**: ユーザー情報が `.local-data/*.json` に平文で保存されている状態は、本番環境では致命的です。

**対策（リリース前必須）**:
- 本番環境は Supabase に切り替え、個人情報カラムは `pgcrypto` で暗号化
- 日次バックアップを外部ストレージ（AWS S3 or Cloudflare R2）に自動保存
- バックアップからのリストア手順を文書化し、1度は実行して動作確認

#### 必須改善 3: **CSRF / XSS / SQL Injection 対策の検証**
**問題**: Next.js のデフォルト保護だけでは、ユーザー入力を受ける全フォームの対策として不十分。

**対策**:
- 全フォームに zod/yup でのスキーマバリデーション
- DOMPurify でHTMLサニタイズ
- Supabase の RLS (Row Level Security) を全テーブルで有効化
- `bun run test:security` のようなセキュリティテストを CI に組み込む

#### 必須改善 4: **認証・権限管理の強化**
**問題**: 現状、工務店ダッシュボードと管理者画面の権限境界が曖昧な箇所がある可能性。

**対策**:
- NextAuth の middleware で全 `/admin/*` と `/dashboard/builder/*` のアクセス権限を厳格チェック
- 工務店アカウントは自社データ以外にアクセスできないことをテストで担保
- 管理者パスワードは本番環境で必ず変更（デフォルト `test1234` などが残っていないか確認）

#### 必須改善 5: **本番環境の死活監視**
**問題**: リリース後に「サイトが落ちた」「APIが応答しない」などが即座に検知できないと、売上を逸失します。

**対策（リリース前必須）**:
- Vercel Analytics または Cloudflare で外形監視
- 主要ページ（/, /diagnosis, /event/xxx）の応答時間を5分ごとに監視
- 応答時間 > 3秒 or エラー率 > 1% でSlack通知

### 🟡 推奨改善 (リリース後1ヶ月以内)

1. **E2Eテストの導入** — Playwright で主要導線（診断→予約→確認メール）をテスト自動化
2. **パフォーマンス計測** — Lighthouse スコアを 90+ に引き上げ（特にLCP / TTFB）
3. **画像最適化** — 施工事例・ルームツアー画像は AVIF or WebP 形式で配信
4. **メール配信の冗長化** — Resend のみに依存すると SPOF。AWS SES と併用
5. **工務店ダッシュボードのログ機能** — 「誰が何を見たか」を監査ログとして保存

### 🔵 将来課題 (6ヶ月以上先)

- マイクロサービス化（コンテンツ・決済・認証の分離）
- BigQuery / Snowflake への分析基盤移行
- モバイルアプリ（React Native）の検討

### 💬 総評
> 「リリース可否：**条件付き可**。ただし『必須改善5項目』のうち、個人情報の暗号化と死活監視は **絶対** にリリース前に対応してください。それ以外はリリース後2週間以内でOKです。
> プロダクトの設計思想は良く、Next.js 16 + Supabase の組み合わせはモダンで将来性があります。ただ、**スタートアップ特有の "勢いで作った" 部分** が散見されるので、リリース前の1-2週間はバグ潰しに専念することを強く推奨します。」

## 💼 専門家C：地域工務店 経営コンサルタントのレビュー

> 「工務店側の現場感覚・実運用可能性を厳しく評価します」

### 🟢 高く評価できる点

#### 1. **工務店ダッシュボードの機能が現場感に合っている**
「管理画面のUIが複雑すぎて使えない」というポータルサイトが多い中、ぺいほーむの工務店ダッシュボードは**シンプルで直感的**です。ITが苦手な50代の工務店社長でも使えるレベル。

#### 2. **料金体系がわかりやすい**
「月額無料＋見学会予約時のみ5万円」は、工務店にとって**意思決定が極めて簡単**です。稟議書を書く必要がないレベル。これはSUUMO/HOME'Sが絶対に真似できない魅力。

#### 3. **成約報酬3%の上限設計は業界にフィットする**
タウンライフ家づくりと同じ3%ですが、「見学会予約時の5万円を差し引く」という設計は**工務店側の心理的負担を軽減**します。上限キャップがあることで、予約が増えすぎて赤字になる不安も払拭できます。

### 🔴 リリース前に必須改善 (4項目)

#### 必須改善 1: **工務店向けのオンボーディング資料が不足**
**問題**: 契約後に工務店が「何から始めればいいか」が明確でありません。ダッシュボードだけでは、工務店側が自走できない可能性があります。

**対策**:
- **契約後7日間の標準オンボーディングフロー** を文書化
  - Day 1: プロフィール登録
  - Day 3: 初回ルームツアー撮影日程調整
  - Day 5: 見学会登録の初期設定
  - Day 7: ダッシュボードの使い方研修（30分オンライン）
- 各ステップに動画マニュアル（5分以内）を用意
- CS担当者が伴走する仕組みを明記

#### 必須改善 2: **見学会予約が入った時の通知とその後のフロー**
**問題**: 見学会予約が発生した時、工務店がどう動けばいいかの標準フローが不明確です。業界的に「通知を見落とす」ことは致命的。

**対策**:
- 予約発生時、**メール + SMS + LINE** の3チャネルで即時通知
- 通知メールには以下を明記:
  - お客様の連絡希望時間・頻度・手段
  - 見学会の目的（じっくり見たい/相談/契約検討）
  - 事前に対応すべきこと（駐車場の案内等）
- ダッシュボードに「未対応」のバッジを表示し、24時間以上未対応の場合は再通知

#### 必須改善 3: **「営業禁止時間」の明文化**
**問題**: スマートマッチで「連絡希望時間」を設定できますが、工務店側がこれを守らなかった場合の対処が不明確。

**対策**:
- 利用規約に「**連絡希望時間外の連絡は禁止**」を明記
- 違反した工務店は 1回: 警告 / 2回: 3ヶ月停止 / 3回: 契約解除
- ユーザーから「しつこい」と報告を受けた工務店は即座に調査開始
- これを契約書と利用規約の両方に明記し、リリース前に工務店に再説明

#### 必須改善 4: **成約報告の運用フロー**
**問題**: 3%成果報酬モデルが正しく機能するには、工務店が「成約しました」と正直に報告する仕組みが必須。現状、報告を工務店任せにするとモラルハザードが発生します。

**対策（タウンライフ家づくりの教訓に学ぶ）**:
- 契約書に「**成約した場合は1週間以内にダッシュボードで報告する義務**」を明記
- 違反時の**違約金条項**を追加（報告漏れ1件につき10万円など）
- ユーザー側にも半年後にアンケート送信：「○○工務店とご契約されましたか？」
- 両方のデータで突合し、食い違いがあれば調査

### 🟡 推奨改善 (リリース後3ヶ月以内)

1. **工務店向けの成功事例共有会** — 先行10社で月1回のオンライン会議を開催し、ノウハウ共有
2. **「工務店目線のQ&A集」の作成** — 実際によくある質問を工務店ダッシュボードに FAQ として常設
3. **月次レポートの自動送信** — 工務店代表にCCで「今月の反響・成約サマリー」を自動メール
4. **動画制作スケジュールの可視化** — ルームツアー撮影の予約管理機能（いつ・何本撮るか）
5. **価格改定アラート** — 坪単価や対応エリアを変更したら、AI診断のマッチング精度に反映するフロー

### 🔵 将来課題 (6ヶ月以上先)

- 工務店横断の**相互紹介ネットワーク**（異なるエリアや強みの工務店同士で、対応できない案件を紹介し合う）
- 工務店向けの**経営ダッシュボード**（粗利・人件費・着工数の管理機能）
- **従業員向け研修コンテンツ**（ぺいほーむ流 接客術など）

### 💬 総評
> 「リリース可否：**条件付き可**。上記『必須改善4項目』のうち、成約報告フローと営業禁止時間の明文化は **絶対にリリース前** に対応してください。これが曖昧だと、半年後に先行10社とトラブルになる可能性があります。
> 工務店側の目線で見れば、ぺいほーむは **近年で最も意欲的で現場感のある** 住宅ポータルです。ただ、**運用の細かい部分（オンボーディング・通知・違反対応）** が詰めきれていないので、リリース前の2週間で運用マニュアルを整備することを強く推奨します。」

2. **成約報告フローの運用整備** (B+C) — 契約書条項・違反時罰則・双方向検証

5. **工務店オンボーディング文書化** (C) — 7日間標準フロー・動画マニュアル
