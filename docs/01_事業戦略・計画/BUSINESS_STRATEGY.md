## 第1章：エグゼクティブサマリー

「ぺいほーむ」は、九州発・平屋市場に特化したAI住宅プラットフォームである。

**事業の定義：**
平屋を検討するユーザーの意思決定を支援し、その意思決定プロセスから得られるデータを蓄積・活用することで、消費者と住宅会社の双方の意思決定精度を向上させるプラットフォーム。

**三面の提供価値：**

| ステークホルダー | 提供価値 |
|-----------------|---------|
| 消費者 | 平屋検討における情報整理・比較・相談・行動支援。AIコンシェルジュが検討フェーズに応じた次のアクションを提案し、最適な住宅会社との出会いまで伴走する |
| 住宅会社 | 単なる反響送客ではなく、意向データ（関心・不安・比較対象・検討フェーズ・温度感）を伴った高精度な見込み顧客接点の提供。将来的にはAI提案支援による商談効率の向上 |
| 運営 | 蓄積される行動データ・会話データ・意向データを基盤として、AI住宅コンシェルジュおよびAI営業支援SaaSへと進化するプラットフォームの構築・運営 |

**現在の事業基盤：**
- YouTube登録者42,841人・公開動画257本の平屋専門コンテンツ資産
- 月間152,228回の視聴（84.8%が新規視聴者）
- 九州エリア12社の工務店パートナーネットワーク
- 62画面のWebプラットフォーム（消費者向け・住宅会社向け・CMS・ダッシュボード）

**目指す姿：**
九州で平屋を検討する人にとっての「意思決定インフラ」となり、平屋市場における情報流通・比較・提案の仕組みそのものを再設計する。

## 第2章：市場背景と課題

### 2.1 平屋市場の拡大

平屋の着工件数は年々増加傾向にあり、特に以下の構造変化が追い風となっている。

- **高齢化社会**：バリアフリー・コンパクト住宅としての平屋需要の増加
- **共働き世帯の増加**：家事動線・回遊動線を重視した効率的な間取りへの関心
- **ライフスタイルの変化**：ミニマルな暮らし・セカンドライフ住宅としての平屋人気
- **九州の特性**：比較的広い土地が確保しやすく、平屋建築に適した市場環境

### 2.2 平屋検討者が抱える課題

| 検討段階 | ユーザーの課題 |
|---------|--------------|
| 情報収集 | 平屋に特化した信頼できる情報源が少ない。一般的な住宅ポータルでは平屋だけを比較しにくい |
| 工務店選び | 平屋が得意な工務店がどこか分からない。展示場を回る時間がない |
| 比較検討 | 間取り・性能・デザイン・価格の比較軸が分からず、判断基準を持てない |
| 相談 | 営業されたくないが専門家に相談したい。自分のペースで検討を進めたい |
| 意思決定 | 一生に一度の買い物で失敗したくない。何を優先すべきか整理できない |

### 2.3 住宅会社が抱える課題

| 課題 | 詳細 |
|------|------|
| 集客コストの高騰 | SUUMO等のポータルサイトの反響単価が上昇。1件あたり8,000〜15,000円 |
| リードの質の低さ | 「とりあえず資料請求」が多く、本気度・検討フェーズが分からない |
| デジタル対応の遅れ | SNS・動画・Webマーケティングのノウハウ・リソースが不足 |
| 商談準備の非効率 | 顧客の関心事・不安・比較対象が事前に分からず、初回商談の精度が低い |

### 2.4 既存サービスの限界

既存の住宅ポータルは「物件を掲載し、問い合わせを送る」仕組みに過ぎない。ユーザーの検討プロセスに寄り添わず、工務店には「誰が何を考えて問い合わせたか」という文脈が伝わらない。

ぺいほーむは、この「送客モデル」の限界を超え、**意思決定支援モデル**へと進化する。

## 第3章：ぺいほーむの提供価値

### 3.1 消費者への提供価値

**「平屋の検討を、ひとりで悩まず、自分のペースで。」**

| 提供価値 | 具体的な機能・体験 |
|---------|-------------------|
| 情報整理の支援 | 平屋特化の動画・記事・事例で、必要な情報を体系的に提供 |
| 比較の支援 | お気に入り保存・比較リスト・閲覧履歴で検討状況を可視化 |
| 相談の支援 | AIコンシェルジュが24時間対応。営業圧力なしで専門知識を提供 |
| 行動の支援 | 検討フェーズに応じた次のアクション（見学会予約・資料請求・相談）を提案 |
| 意思決定の支援 | ユーザーの関心・予算・家族構成に合った工務店を根拠付きで推薦 |

### 3.2 住宅会社への提供価値

**「反響の数ではなく、反響の質を上げる。」**

| 提供価値 | 具体的な機能・体験 |
|---------|-------------------|
| 高精度リード | 意向データ付き：関心タグ・不安要素・比較対象・検討フェーズ・温度感 |
| 商談前の意向要約 | AIが会話ログ・行動履歴から「この人が何を求めているか」を自動要約 |
| 提案支援 | 「この顧客には○○を訴求すべき」というAI提案を商談前に提供（将来） |
| 集客代行 | ルームツアー動画撮影・SNS運用・Web制作で認知〜問い合わせを支援 |
| 営業改善データ | 成約者の閲覧傾向・相談カテゴリ・検討期間のデータで営業改善を支援（将来） |

### 3.3 価値創出の構造

```
ユーザーが平屋を検討する
  ↓ 動画を観る・記事を読む・AIに相談する・比較する・保存する
行動データ・会話データが蓄積される
  ↓
AIが意向を推定する（関心・不安・フェーズ・温度感）
  ↓
ユーザーには：次のアクション提案・パーソナライズ推薦
住宅会社には：意向データ付きリード・商談前要約
  ↓
両者の意思決定精度が上がる
  ↓
成約率が上がる → 住宅会社の支払意欲が上がる → プラットフォームの収益が上がる
```

この構造の核心は、**ユーザーの検討行動そのものがデータ資産になり、サービスの価値を高め続ける**点にある。

## 第5章：競争優位性

### 5.1 5つの競争優位性

| # | 優位性 | 内容 | 模倣困難性 |
|---|--------|------|-----------|
| 1 | 平屋特化の一次情報資産 | 257本の平屋ルームツアー動画。実際の家を撮影・取材した一次情報であり、ネット上の二次情報とは質が異なる | 高：撮影には工務店との関係構築と現場アクセスが必要。後発が同量を揃えるには数年かかる |
| 2 | 平屋検討層との文脈理解 | 42,841人の登録者が「何に反応し、何を視聴し、どんなコメントを残したか」という視聴行動の知見。平屋検討者の関心・不安・比較行動に関する暗黙知 | 高：YouTube運営を通じて年単位で蓄積されたもの。データとして取得しようとしても同じ文脈は得られない |
| 3 | 九州工務店ネットワーク | 12社のパートナー工務店との信頼関係。取材・撮影を通じた各社の強み・弱み・顧客層の理解 | 中：関係構築は時間がかかるが、人的営業で追随可能 |
| 4 | データ蓄積基盤の先行者優位 | Web上の閲覧・相談・予約・資料請求・会話ログ・比較履歴を構造化して蓄積するプラットフォーム。早期に稼働するほどデータ量で差がつく | 高：データの蓄積量は時間に比例し、後発は追いつけない |
| 5 | AI推薦・提案支援への接続可能性 | 蓄積データを基に、ユーザーへの推薦精度向上、住宅会社への意向要約・提案支援に展開できる。単なるメディアやポータルにはない進化経路 | 高：データ基盤がなければAIは機能しない。基盤+データ+ドメイン知識の組み合わせ |

### 5.2 既存競合との差別化

**重要な市場変化（2026年1月）：** 日本で唯一の平屋特化ポータルであった「e平屋net」（Lib Work社運営、2020年開設）が、2026年1月31日をもって閉鎖された。これにより、**日本に平屋専門のプラットフォームは1つも存在しなくなった。ぺいほーむが日本初かつ唯一の平屋特化AIプラットフォームとなる。**

e平屋netの閉鎖は、「物件を掲載するだけ」の従来型ポータルアプローチでは平屋市場で持続的な価値を創出できないことを証明している。ぺいほーむのYouTube一次情報 × AI × データ蓄積という差別化アプローチの正しさを裏付ける市場事実である。

| 競合類型 | 代表例 | ぺいほーむとの違い |
|---------|--------|-------------------|
| 総合住宅ポータル | SUUMO / HOME'S | 平屋特化ではない。動画コンテンツがない。ユーザーの検討プロセスを支援しない。送客モデル |
| 平屋特化ポータル（閉鎖済） | e平屋net（Lib Work社）※2026/1/31閉鎖 | 物件掲載型のみで閉鎖。YouTube・AI・データ蓄積の統合基盤がなかった。掲載型アプローチの限界を示す事例 |
| 住宅系YouTuber | 個人チャンネル | 動画のみ。Web・CRM・AIの統合基盤がない。データを蓄積・活用する仕組みがない |
| 住宅コンサル | 対面型サービス | スケールしない。デジタルデータが残らない。属人的 |
| AI住宅サービス | 汎用AIチャット | 平屋の一次情報がない。ドメイン知識が浅い。ユーザー行動データがない |

**ぺいほーむの本質的な差別化：** 平屋の一次情報 × ユーザー行動データ × AI活用。この3つを統合して持つプレイヤーは現時点で存在しない。唯一の平屋特化ポータルであったe平屋netが閉鎖された今、ぺいほーむは日本で唯一の平屋専門プラットフォームであり、この事実は工務店・投資家に対する強力な訴求ポイントとなる。

#### Phase 1：データ基盤構築（最優先）

**目的：** 将来のAI活用に必要なデータを取得・蓄積する仕組みを構築する。

| 実装項目 | なぜ最優先か |
|---------|-------------|
| ユーザー識別子の統一 | 匿名ユーザーの行動を、会員登録後に紐付けるために必要。これがないと行動データがバラバラになる |
| 行動イベント取得 | ページ閲覧・動画視聴・記事読了・お気に入り追加等のイベントを記録。AIが学習するデータの根幹 |
| チャットログの構造化保存 | 会話内容・推定意図・抽出タグ・要約を保存。意向推定とリード品質向上の材料 |
| リードと行動データの紐付け | 「この人がなぜ問い合わせたか」の文脈を住宅会社に渡すため |
| GA4カスタムイベント計測 | ファネル分析・コンテンツ評価・CV率計測の基盤 |
| CMS構造化データの入力項目追加 | 工務店・物件・記事にAI推薦用のタグ（デザインテイスト・性能・向いている顧客像等）を追加 |

#### Phase 2：AI活用開始

**目的：** 蓄積されたデータを使い、推薦・パーソナライズ・意向データ付きリードを実現する。

| 実装項目 | 内容 |

## 第8章：データ戦略 / AI戦略

### 8.1 データ戦略の基本方針

**「すべてのユーザー接点を、AIの学習データにする。」**

ぺいほーむの事業価値の源泉は、ユーザーが平屋を検討する過程で自然に生成されるデータにある。動画を観る、記事を読む、AIに相談する、お気に入りに追加する、工務店を比較する——これらの行動すべてが、推薦精度を高め、リード品質を上げ、プラットフォームの価値を押し上げるデータとなる。

### 8.2 取得するデータと活用用途

| データ種別 | 取得方法 | 活用用途 |
|-----------|---------|---------|
| 閲覧履歴 | page_view / video_viewイベント | 関心分野の推定、コンテンツ推薦 |
| お気に入り | ユーザーの明示的な保存操作 | 検討中の物件・工務店の把握、推薦材料 |
| 比較履歴 | 比較リストへの追加操作 | 比較軸・判断基準の推定 |
| チャットログ | AIコンシェルジュとの会話 | 意向抽出・不安把握・意図推定・要約生成 |
| フォーム送信 | 相談・資料請求・予約フォーム | リード化、流入文脈の記録 |
| ユーザープロフィール | 段階的なヒアリング・自己申告 | パーソナライズ、セグメント分析 |
| エンゲージメント | 滞在時間・スクロール深度・クリック | コンテンツ品質評価、関心度推定 |

### 8.3 AI進化の段階

```
Phase 1: データを貯める
  ・行動イベントの記録
  ・チャットログの保存
  ・ユーザー識別子の統一

Phase 2: データを使い始める
  ・閲覧傾向からの簡易推薦
  ・チャットログからの意向抽出・タグ付け
  ・リードへの行動文脈付与
  ・検討フェーズの推定

Phase 3: AIが提案する
  ・ユーザーごとのnext best action推定
  ・工務店ごとの提案最適化
  ・商談前要約の自動生成
  ・成約予測モデル
```

### 8.4 データ蓄積のフライホイール

```
ユーザーが増える → 行動データが増える → 推薦精度が上がる
  ↑                                          ↓
  └── リード品質が上がる ← 工務店の満足度が上がる ← 成約率が上がる
```

このフライホイールが回り始めると、データ量と推薦精度が相互に強化し合い、後発プレイヤーとのデータ格差が拡大し続ける。

### 8.5 プライバシーとデータ倫理

- 行動データの取得はユーザーへの明示的な通知と同意に基づく
- 個人を特定するデータと行動データは分離して管理する
- 住宅会社に提供するのは集計・匿名化されたインサイトであり、個人の生データではない
- ユーザーは自身のデータの閲覧・削除を要求できる

### 9.1 集客戦略

| チャネル | 役割 | 施策 |
|---------|------|------|
| YouTube | 認知・集客の主軸 | 平屋ルームツアーの継続制作。SEO対応タイトル。動画内でWeb誘導 |
| SEO | 検索からの流入 | 「平屋 + エリア名」「平屋 間取り」等のキーワードで記事を展開 |
| SNS | 拡散・エンゲージメント | Instagram/TikTokでショート動画。Threadsでコミュニティ形成 |
| LINE | CRM・再訪促進 | 友だち追加→検討フェーズに合わせた情報配信 |

### 9.2 ユーザー獲得ファネル

```
認知 ── YouTube動画 / SNS / SEO検索
  ↓ [page_view / video_view イベント記録]
興味 ── Web物件詳細 / 記事 / セミナー
  ↓ [article_read / builder_detail_view 記録]
検討 ── お気に入り保存 / 比較 / AIチャット相談
  ↓ [favorite_add / comparison_add / chat_start 記録]
行動 ── 無料相談 / 資料請求 / 見学会予約
  ↓ [consultation_request / catalog_request / reservation_submit 記録]
成約 ── 工務店紹介 → 面談 → 契約
  ↓
推薦 ── お客様の声投稿 → 新規集客へ循環
```

各ステップで行動イベントを記録し、ファネルの歩留まりを計測・改善する。

### 9.3 パートナー拡大戦略

| 段階 | パートナー数 | エリア |
|------|------------|--------|
| 現在 | 12社 | 鹿児島市中心 |
| 6ヶ月後 | 25社 | 鹿児島県全域 + 福岡市 |
| 1年後 | 50社 | 九州主要都市（全県対象） |
| 2年後 | 100社 | 九州全域 |

**地域戦略の補足：** e平屋net（Lib Work社・熊本拠点）が2026年1月31日に閉鎖されたため、九州全域において平屋特化の競合プラットフォームは皆無となった。ぺいほーむは初日から福岡・熊本・佐賀・大分を含む九州全県をターゲットとすることが可能である。

### 9.4 リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| YouTube依存 | 高 | SEO・SNS・LINE等への分散。Web上のデータ蓄積でYouTubeに依存しない価値基盤を構築 |
| データが溜まらない | 高 | Phase 1でデータ基盤を最優先実装。ユーザーDB/行動取得/チャットログをMVPの最重要項目に |
| 工務店の離脱 | 中 | 成果報酬型で低リスクを訴求。意向データ付きリードで「質」の高さを実感してもらう |
| 競合の模倣 | 中 | データ蓄積による先行者優位。平屋特化の一次情報は短期間では模倣困難。e平屋net（Lib Work社）の閉鎖（2026/1/31）は、掲載型アプローチでは平屋特化市場を維持できないことを証明しており、ぺいほーむのYouTube+AI+データ型アプローチが差別化として有効 |
| 個人情報リスク | 高 | Supabase RLS + 行動データと個人データの分離 + 監査ログ + 同意管理 |
