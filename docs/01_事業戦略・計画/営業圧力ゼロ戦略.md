# 営業圧力ゼロ戦略（Anti-Pressure Strategy）

**作成日**: 2026-04-11
**対象**: ぺいほーむ v4.0 プラットフォーム戦略
**位置づけ**: REQUIREMENTS.md / BUSINESS_STRATEGY.md の補完戦略ドキュメント

---

## 1. 課題認識

ユーザーヒアリングから「しつこい営業や営業を受けることが嫌」という声が頻繁に出ている。一方で、住宅購入という高額かつ情報非対称な意思決定において、工務店との商談は **構造的に避けて通れない** プロセスである。

この「顕在的ニーズと構造的現実のズレ」を放置すると、

- ユーザーは情報収集段階で離脱し、会員登録・見学会予約に至らない
- 工務店も質の悪いリードを追い続け、双方が疲弊する
- ぺいほーむは「やはり他ポータルと同じ」と評価されてしまう

という悪循環に陥る。

---

## 2. 痛みの分解

「営業が嫌」を因数分解すると、実際には以下の6つの痛みが混ざっている。

| # | 痛み | 具体的な症状 | 性質 |
|---|------|------------|------|
| 1 | コントロール喪失 | 「1回連絡先を渡したら、あとは向こう主導」 | **顕在** |
| 2 | 時間の侵食 | 夜・週末に電話、反復連絡 | **顕在** |
| 3 | 情報非対称性 | 何を聞けばいいか分からないまま話が進む | **顕在** |
| 4 | 決断圧力 | 「今月中なら〇〇万円引き」等の時限プレッシャー | **顕在** |
| 5 | 反復説明疲れ | 何社にも同じ家族構成・予算を説明する | **顕在** |
| 6 | 断る気まずさ | ここまで時間を取らせたのに断りにくい | **潜在** |

**結論**: 6番目の「断る気まずさ」だけが潜在的で、残り1〜5はすべて顕在的な不満。機能で分解すれば解決可能な範囲である。

---

## 3. 既存サービスが解決できない理由

| 既存サービス | 解決アプローチ | 限界 |
|--------------|--------------|------|
| SUUMO / HOME'S | 問い合わせを工務店に送るだけ | ユーザー保護の仕組みがない |
| タウンライフ | 一括資料請求 | 送客後の連絡を制御できない |
| 無料相談サイト | 「しつこい営業なし」と標榜 | 強制力がなく、遵守は工務店任せ |
| 匿名ブラウジング（一般） | 連絡先を渡さない | パーソナライズ・診断結果保存ができない |

---

## 4. ぺいほーむが解決できる理由（独自ポジション）

ぺいほーむは **会員アカウント × AI診断 × 中立プラットフォーム** という独自の3点セットを持っている。このため、以下が技術的に可能である。

1. **ユーザーと工務店の間に「AI仲介者」を置ける** — 直接接触前にバッファできる
2. **連絡条件を「会員プロフィールの一部」として設定させ、工務店に強制できる** — 契約条件として守らせる
3. **ユーザーの「検討フェーズ」を動的に追跡し、工務店側にリアルタイム共有できる** — 積極性を自動調整
4. **工務店側のダッシュボードから直接「見込み客の現在の温度感・連絡OK時間帯」を表示できる** — データで守る

## 6. 推奨する組み合わせ：「Anti-Pressure Pack」

単独機能より、以下の3点セットが相乗効果で最強である。

### コア① 連絡条件設定

ユーザーがマイページで以下を設定。見学会予約・問い合わせ時に工務店へ自動送信され、工務店は契約上遵守義務を負う。

| 項目 | 選択肢 |
|------|--------|
| 連絡頻度 | 週1回まで / 月1回まで / 当方から連絡するのみ |
| 連絡手段 | メール / LINE / 電話OK / 電話NG（複数選択可） |
| 連絡時間帯 | 平日昼間 / 平日夜 / 週末 / いつでもOK |
| 連絡目的 | 情報提供のみ / 質問回答のみ / 商談提案OK |
| 追加メモ | 自由記述欄 |

### コア② 検討フェーズ宣言

マイページで自分の現在フェーズを明示。工務店側ダッシュボードにリアルタイム反映。

- **Phase A 情報収集中**: 連絡は求めたときだけ
- **Phase B 比較検討中**: 週1ペースまでOK
- **Phase C 具体検討中**: 週2-3 OK、見学会歓迎
- **Phase D 契約前**: 積極的な連絡歓迎

### コア③ 工務店側の遵守義務（規約化）

上記を単なる要望ではなく **契約条件** として明示。違反があればプラットフォーム退出。これが肝で、ユーザーに「ぺいほーむ経由なら安心」と言える根拠になる。

## 7. Phase 2 以降の拡張機能

### AI仲介質問機能

工務店個別ページに「匿名で質問する」ボタン。AI経由で質問を工務店に投げ、返答をAIが整形してユーザーに戻す。ユーザーが満足したタイミングで「連絡先を開示する」ボタンを押すと実名化される。

実装イメージ:

```
User → [匿名質問] → ぺいほーむAI → 工務店
                                     ↓
User ← [AIが整形] ← ぺいほーむAI ← 回答
  ↓
「この工務店とさらに話したい」→ 実名開示 → 工務店へ連絡先送付
```

### 共通プロフィール自動送信

AI診断結果と連絡条件を「ユーザー名刺」としてPDF化、見学会予約時に自動送信。何度も家族構成・予算を説明する必要がなくなる。

含まれる情報:

- 家族構成（自動入力）
- 予算レンジ（自動入力）
- 希望エリア（自動入力）
- 検討フェーズ（自動入力）
- AI診断タイプ（自動入力）
- 連絡条件（自動入力）
- 特記事項（ユーザー任意記入）

### 見学会モード選択

見学会予約時にユーザーが選択:

- **体感モード**: 実物を見るだけ。商談トークNG
- **相談モード**: 具体的な間取り・予算の相談OK
- **契約検討モード**: 詳細見積もり・契約の話OK

工務店側は選択モードに応じて対応を変える義務を負う。

## 9. KPI 設計

### 効果測定指標

| 指標 | 計測方法 | 目標（3ヶ月後） |
|------|---------|--------------|
| 連絡条件設定率 | 会員登録者のうち、条件を設定した割合 | 60%以上 |
| 見学会ドロップオフ率 | 予約後キャンセルの割合 | 15%以下 |
| 「しつこい」クレーム件数 | ユーザーからの苦情報告 | 月1件以下 |

| 会員登録→見学会予約転換率 | AI診断経由の転換率 | 40%以上 |

### ユーザーフィードバック収集

- 見学会参加後の満足度アンケート（営業体験への評価項目あり）
- 非アクティブ会員への離脱理由調査
- 工務店からの「ユーザーの温度感」レポート

## 10. リスクと対応

| リスク | 対応策 |
|--------|--------|
| 工務店が条件を守らない | プラットフォーム退出ルール + 違反報告システム |
| ユーザーが条件を設定しすぎて営業が機能しない | デフォルトを「連絡OK」に設定 + 過度な制限に対する警告 |
| 機能が複雑になり離脱 | 3-stepウィザードでの簡単設定 |
| 工務店側のオンボーディング負荷 | マニュアル提供 + 月次レポートで遵守状況可視化 |

---

## 11. 結論

ユーザーの「営業が嫌」という声は、顕在的な6つの痛みに分解できる。ぺいほーむは中立プラットフォームとして、**連絡条件設定 × 検討フェーズ宣言 × 工務店遵守義務** の3点セットでこれらを解決できる。

この戦略は単に「ユーザーの不満を解消する」だけでなく、

- **差別化**: SUUMOにはできない「ユーザー保護」を前面に打ち出せる
- **信頼**: 「ぺいほーむ経由なら安心」というブランド価値
- **成約率向上**: 無駄な営業の削減により、両者が本気の案件に集中できる

を同時に実現する。
