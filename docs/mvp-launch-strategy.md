# 5月リリース MVP戦略 ─ 選択と集中

**作成日**: 2026/04/11
**リリース日**: 2026/05/01 (目標)
**基本方針**: **「核のループ」以外は一旦隠す or 簡素化する**

---

## 🎯 この戦略の核心

> **1本のコアフローだけを完璧に動かしてリリースする。残りは「見せない」「作らない」「後回しにする」のどれかに分類する。**

今のぺいほーむは **131ページ、51ユーザー向け画面、15種類のCMSコンテンツ**を抱えています。この規模は、通常3-5名の開発チームが1年かけて作るボリュームです。**これをそのままリリースするのは非現実的**です。

---

## 🔑 「コアループ」を1つに定める

リリース後の3ヶ月間、私たちが最大化したい行動は**これ1つだけ**です:

```
┌────────────────────────────────────────────────┐
│                                                │
│   YouTube ─▶ TOP ─▶ 工務店ページ ─▶ 見学会予約    │
│                       │                          │
│                       └▶ AI診断 ─▶ 工務店ページ   │
│                                                │
└────────────────────────────────────────────────┘
```

**この1本のループが動かない限り、事業は成立しません。** 逆に言えば、このループさえ回れば、他の機能は後から追加できます。

## 🛠 "HIDE" の技術的な実現方法

**ファイル自体は削除しない**（後で復活させやすくするため）。代わりに以下の3つのガード層で隠します：

### ガード層 1: ナビゲーション削除
以下のファイルから該当リンクを削除：
- `src/components/layout/Header.tsx`
- `src/components/layout/Footer.tsx`
- `src/components/layout/MobileNav.tsx`
- `src/app/sitemap.ts`

### ガード層 2: ミドルウェアでの404返却
`src/middleware.ts` に隠す経路を列挙し、アクセス時は404を返す：

```ts
const HIDDEN_PATHS = [
  '/articles', '/news', '/interview', '/magazine', '/webinar',
  '/sale-homes', '/lands', '/features', '/simulator', '/area',
  '/builders/compare', '/builders/contact',
  '/mypage/catalog', '/mypage/questions', '/mypage/feedback',
  '/biz/service', '/biz/ad', '/biz/partner',
  '/biz/articles', '/biz/news', '/biz/webinar',
];
```

### ガード層 3: robots.txt で検索エンジンから除外
`public/robots.txt` に `Disallow: /articles` などを追加。

→ **後で復活させたい時は、この3層から外すだけで即復活できる**。コードは温存される。

## 💡 この戦略の5つのメリット

### ① バグの総量が激減する
131画面 → 29画面 = **バグ発生率 78%減**
リリース後の緊急修正対応が現実的な範囲に収まる。

### ② 運用負荷が激減する
15種類のCMS更新 → 3種類 = **コンテンツ運用負荷 80%減**
現状のチーム規模(3-4名)で運用できる。

### ③ ユーザーの迷いがなくなる
「どこを見たらいいかわからない」が消える。
AI診断 → 工務店 → 見学会予約 の1本道が明確になる。

### ④ 月次レビューがシンプルになる
見るべきKPIが絞られる：
- 見学会予約件数
- AI診断完了数
- 工務店別成約数
- NPS

### ⑤ 成功体験を積み上げやすい
コアループが1週間で10予約を生めば、**次フェーズの判断が明確**になる。
反対に、すべてを並行で動かすと「何が機能しているか」が測れない。

## ⚠️ 今やってはいけない3つのこと

### ❌ 1. 機能追加の追加開発
新しい機能を足したくなるが、リリース後1ヶ月は**追加開発禁止**。バグ修正と運用改善のみ。

### ❌ 2. 優先順位の曖昧化
「せっかく作ったから全部出したい」という感情を捨てる。隠したコードは資産として保存されている。

### ❌ 3. 先行10社全員への同一説明
10社の中でも**動画撮影に前向きな3社を先行**させる。残り7社は様子を見ながら段階的にオンボーディング。

## 🎯 結論

> **"作ったものを全部出す" のではなく、"動かしたいループだけ出す"。**
>
> 5月1日のリリースで勝負するのは **22画面** だけで十分です。
> 残り 109画面はコードとして温存し、ユーザーの反応を見ながら月1-2機能のペースで復活させます。
>
> これがスタートアップの鉄則であり、先行10社との信頼を守り、
> 事業を着実にグロースさせる唯一の道です。
