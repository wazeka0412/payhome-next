# 運用ルール

---

## ステータス定義

| ステータス | 意味 | 色 |
|-----------|------|-----|
| 未着手 | まだ着手していない | ⬜ グレー |
| 進行中 | 作業中 | 🟡 黄色 |
| レビュー待ち | 作業完了、確認待ち | 🟠 オレンジ |
| 完了 | 完了条件を満たした | 🟢 緑 |
| ブロック中 | 依存タスク未完了で着手不可 | 🔴 赤 |

---

## 優先度定義

| 優先度 | 意味 | 色 |
|--------|------|-----|
| P0 | リリース前必須 — 未完了ならリリース延期 | 🔴 赤 |
| P1 | リリース直後必須 — 1週間以内に完了 | 🟡 黄色 |
| P2 | Phase1運用中 — 定常的に実施 | 🔵 青 |
| P3 | Phase2準備 — 6月以降着手 | ⬜ グレー |
| Backlog | 後回し — Phase3以降検討 | - |

## 更新ルール

| タイミング | 担当 | アクション |

| 毎朝9:00 | エンジニア | Sentry/Vercel確認後にステータス更新 |

## 朝会での使い方

1. ダッシュボードを開き、P0→P1の順にステータス確認
2. 「ブロック中」タスクの依存解消を議論
3. 当日完了予定タスクの担当確認
4. 新規課題があれば即タスク追加

## Notionデータベースのビュー設定（推奨）

### ビュー1: ボードビュー（ステータス別）
- グループ: ステータス
- フィルタ: なし
- ソート: 優先度 昇順 → 期限 昇順

### ビュー2: テーブルビュー（全タスク）
- フィルタ: なし
- ソート: 期限 昇順
- 可視列: ID, タスク名, 優先度, 担当者, 期限, ステータス

### ビュー3: タイムラインビュー
- 開始: 期限（日付プロパティ）
- グループ: フェーズ

### ビュー4: フィルタビュー（P0のみ）
- フィルタ: 優先度 = P0
- ソート: 期限 昇順

### ビュー5: 担当者別
- グループ: 担当者
- フィルタ: ステータス ≠ 完了

| 優先度 | セレクト | P0, P1, P2, P3, Backlog |
| 担当者 | セレクト | エンジニア, PM, 営業, 代表, 経理, 全員 |
| 期限 | 日付 | - |
| ステータス | セレクト | 未着手, 進行中, レビュー待ち, 完了, ブロック中 |
| 親タスク | テキスト | - |
| 依存タスク | テキスト | - |
| 完了条件 | テキスト | - |
| 備考 | テキスト | - |
